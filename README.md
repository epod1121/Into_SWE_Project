# Into to Software Engineering Project

General Planner

Overview

The General Planner is a browser-based deadline tracking system designed to help students and professionals manage schedules. The project addresses the need for a central system to track multi-stage deadlines—such as discussion board posts that require initial entries and subsequent responses—which are often poorly defined or easy to overlook. Users can manage a personal calendar and receive automated browser notifications as deadlines approach.



Key Features


Event Management: Full capabilities for calendar events, including titles, descriptions, and due dates.

Automated Notifications: Browser alerts sent 24 hours and 1 hour before a scheduled event.

Responsive UI: An interface that allows users to view and manage their calendar on any device with any browser.

Categorization: Ability to use custom tags (e.g., Work, School, Personal) to organize upcoming events.



Technology Stack


The project is made using a simple website stack to makes sure it works without heavy frameworks

Frontend: HTML and CSS for a functional and simple interface.

Logic Layer: JavaScript for handling event listeners, internal clock logic, background color, and notifications.

Backend/Storage: SQL database for storage of user event data.



System Architecture


The application follows a three-tier architecture:

Frontend: Manages the calendar view and processes user input.

Logic (JavaScript): Processes deadlines, background color, and triggers notifications.

Database (SQL): Stores event details



Installation and Requirements


Platform: Accessible through any modern browsers such as Chrome, Safari, and Edge.

Connectivity: Requires an internet connection to see stored data and receive notifications.

Manual Entry: Events must be entered manually. The current version does not support automatic syncing with external calendars or AI-based suggestions.
